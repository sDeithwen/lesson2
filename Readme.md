## Практическая работа №2
Цель: Собрать свой первый Docker-образ для простого Python-приложения, запустить контейнер и понять базовые инструкции Dockerfile.
Проект: https://github.com/sDeithwen/lesson2 

Приложение:
- app.py запускается и периодически выводит текст в консоль.
- Приложение работает постоянно и не должно завершаться сразу.
### Что нужно сделать
Нужно собрать Docker-образ для этого приложения и запушить его Docker Hub.
### Обязательные требования
- Использовать образ с Python, например python:3.11.
- Внутри образа должна быть установлена библиотека из файла requirements.txt через команду:
```RUN pip install --no-cache-dir -r requirements.txt
```
- В образе есть nano
- У образа должен быть тег.
- В контейнере приложение должно находиться в папке /app.
- Запуск приложения должен происходить через:
```CMD ["python", "app.py"]
```
- В контейнере должен быть установлен nano.
### Базовая сдача
Для базовой сдачи должно быть выполнено следующее:
- Образ собирается без ошибок.
- Контейнер запускается и не завершается сразу.
- В логах контейнера периодически появляется сообщение приложения.
- В образе установлен nano.
- Ученик может показать свой Dockerfile и коротко объяснить каждую строку.
### Основные инструкции Dockerfile
- FROM — базовый образ.
- WORKDIR — рабочая папка для всех следующих команд.
- COPY — копирование файлов в образ.
- RUN — выполнение команд при сборке образа.
- ENV — переменные окружения.
- EXPOSE — документация порта приложения.
- CMD — команда по умолчанию при запуске контейнера.

8. Если база готова — перейти к заданиям со звёздочкой.
### Шпаргалка по командам
### Сборка образа
```docker build -t my-app:v1 .
```
### Запуск контейнера
```docker run --name my-app my-app:v1
```
### Запуск в фоне
```docker run -d --name my-app my-app:v1
```
### Просмотр запущенных контейнеров
```docker ps
```
### Просмотр логов контейнера
```docker logs -f my-app
```
### Остановка контейнера
```docker stop my-app
```
### Удаление контейнера
```docker rm my-app
```
### История слоёв образа
```docker image history my-app:v1
```
### Список локальных образов
```docker images
```
### Зайти внутрь контейнера
```docker exec -it my-app bash
```
### Добавить образу новое название
```docker tag my_image:1.0 sdeithwen/my_image:1.0```
### Команды для работы с Docker Hub
```docker login
docker push sdeithwen/my_image:1.0
```
